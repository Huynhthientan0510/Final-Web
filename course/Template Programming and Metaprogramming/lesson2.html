Lesson 2: Function and Class Templates

Introduction

Templates can be used not only for functions but also for classes. In this lesson, you’ll learn about function templates and class templates in greater detail. While function templates allow you to create generic functions, class templates enable you to create classes that work with multiple types. Understanding the difference and how to use them both is key to mastering C++'s generic programming features.

Purpose of the Lesson

The goal of this lesson is to explore function and class templates in C++. By the end of this lesson, you should be able to:

  • Write and use both function and class templates.
  • Understand the benefits of each type of template.
  • Apply class templates to real-world programming scenarios like generic data structures.

Detailed Explanation

A function template provides a way to create a function that works with any data type. You’ve already seen simple examples of function templates in the previous lesson, but now we’ll extend those concepts.

Let’s look at a slightly more complex example that works with two different types:

// Example C++ code
template <typename T, typename U>
T multiply(T a, U b) {
	return a * b;
}

Explanation:

  • Multiple Template Parameters : The template above has two template parameters: T and U . This means that a and b can be of different types. For example, you can multiply an int by a double .
  • Flexibility : This template is more flexible than one with a single type because it allows mixing types. You could call multiply(3, 5.5) and the template would automatically handle the different types.

Now, let’s explore class templates .

A class templates works similarly to a function template, but it allows you to create generic classes. This is especially useful for creating data structures like lists, stacks, and queues that can work with any data type. 


template <typename T>
class Box {
	T value;
public:
	Box(T val) : value(val) {}
	T getValue() const { return value; }
	void setValue(T val) { value = val; }
};

Explanation:

  • The Box class is a template that can store any type T. You can create an instance of Box<int>, Box<double>, or even Box<string>.
  • The class has a constructor that initializes the value of type T , a getter method getValue() that returns the stored value, and a setter method setValue() to update the value.

Example usage:


int main() {
	Box<int> intBox(123);
	cout << "Value in intBox: " << intBox.getValue() << endl;
	intBox.setValue(456);
	cout << "Updated Value in intBox: " << intBox.getValue() << endl;

	Box<string> strBox("Hello");
	cout << "Value in strBox: " << strBox.getValue() << endl;
	strBox.setValue("World");
	cout << "Updated Value in strBox: " << strBox.getValue() << endl;

	return 0;
}

Explanation:

  • In main() , two Box objects are created: one for int and one for string .
  • You can see that the same class works with both data types, which is the main advantage of class templates: code reuse and type safety.
  • Code Example: A Generic Pair Class

    • Another common example of a class template is a generic Pair class that stores two values: 

    
template 
class Pair {
	T1 first;
	T2 second;
public:
	Pair(T1 a, T2 b) : first(a), second(b) {}
	T1 getFirst() const { return first; }
	T2 getSecond() const { return second; }
	void setFirst(T1 a) { first = a; }
	void setSecond(T2 b) { second = b; }
};

    Explanation:

    • The Pair class template takes two template parameters, T1 and T2 , meaning that the two values first and second can be of different types.
    • This class can be used to create pairs of any types, such as Pair>int, double>, Pair>string, int>, or even Pair>char, bool>.

    Example Usage:

    
int main() {
	Pair p1(10, 20.5);
	cout << "First: " << p1.getFirst() << ", Second: " << p1.getSecond() << endl;

	Pair p2("Age", 30);
	cout << "First: " << p2.getFirst() << ", Second: " << p2.getSecond() << endl;

	return 0;
}

    Explanation:

    • In main() , the Pair class template is used to store a pair of an integer and a double, and then a string and an integer..
    • The template automatically generates the correct class definitions for each type combination, allowing you to store and retrieve different types easily.
  • Benefits of Class Templates
    1. Reusability :Write a class once, use it for many types.
    2. Reusability :The compiler ensures that only valid operations are performed on the types used in the template
    3. Reusability :Templates are evaluated at compile-time, leading to optimized code.

    Best Practices and Tips

    • Use Class Templates for Data Structures :Templates are perfect for creating generic data structures like linked lists, queues, and stacks that can store any type of data.
    • Leverage Multiple Template Parameters :If your class needs to handle multiple types, don’t hesitate to use more than one template parameter
    • Limit Template Use in Complex Applications :While templates are powerful, avoid using them in overly complex scenarios where polymorphism or other techniques might be more appropriate.

    Common Mistakes and Pitfalls

    • Incorrect Template Parameter Usage :Be careful to ensure that all template parameters are used correctly within the class. If a type doesn’t support the operations in your template, you will get confusing compiler errors.
    • Over-Templating : Resist the urge to template everything. Sometimes, simple overloading or inheritance is more appropriate, especially for smaller or more specialized tasks.

    Real-World Applications

    The C++ Standard Template Library (STL) is built entirely on templates. Classes like std::vector , std::pair , and std::map are all class templates. These classes are capable of working with any data type, which is why the STL is so versatile and powerful.

    Let’s look at an example with std::pair : 

    
#include <utility>
#include <iostream>
using namespace std;

int main() {
	pair<int, string> student(1, "John");
	cout << "ID: " << student.first << ", Name: " << student.second << endl;

	return 0;
}

    Explanation:

  • The std::pair template allows you to create a pair of values of any types. Here, a pair of an integer (student ID) and a string (student name) is created. The first and second members of the pair hold the respective values.

    • This kind of flexibility makes templates invaluable for generic programming.

    Advanced Concepts

    • Partial Specialization :While template specialization involves providing a specific implementation for a particular type, partial specialization allows you to specialize templates for a subset of template parameters. This is a more advanced topic that allows greater control over the behavior of templates.
    • Non-Type Template Parameters :Templates can also accept non-type parameters, such as integer values or pointers, allowing for even more flexibility in defining templates.

    Summary

    Function and class templates are a cornerstone of generic programming in C++. They enable developers to write flexible, reusable code that works with multiple data types without duplication. Function templates are ideal for generic algorithms, while class templates are commonly used for data structures that can handle various types of data.
