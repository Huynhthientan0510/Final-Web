Lesson Title: Dynamic Memory Allocation (new, delete)

Introduction

Dynamic memory allocation in C++ allows the allocation of memory during runtime, providing flexibility in managing memory efficiently. The `new` and `delete` operators are essential tools for managing this dynamic memory, allowing developers to create and destroy objects as needed.

Purpose of the Lesson

The purpose of this lesson is to understand how to use `new` and `delete` for dynamic memory allocation and deallocation. Mastery of these operators is crucial for creating scalable applications and managing memory efficiently.

Detailed Explanation

  • Dynamic Memory Allocation : Memory allocation at runtime using the new operator; declares a pointer to an integer.
  • new Operator: Allocates memory from the heap and returns a pointer to the allocated space.
  • delete Operator : Deallocates memory previously allocated with `new`, freeing up resources.
  • Array Allocation : new can also allocate memory for arrays.
  • Memory Leaks :Occur when allocated memory is not properly deallocated, leading to wasted resources.

Code Examples


#include 
int main() {
	// Dynamic allocation of a single integer
	int* ptr = new int(10); // Allocate memory for an integer and initialize it to 10.
	std::cout << "Value: " << *ptr << std::endl; // Output the value pointed to by ptr.
	delete ptr; // Deallocate the memory to avoid memory leaks.
	// Dynamic allocation of an array
	int size = 5;
	int* arr = new int[size]; // Allocate memory for an array of 5 integers.
	// Initialize and print array elements
	for (int i = 0; i < size; i++) {
	arr[i] = i * 10;
	std::cout << "Array element " << i << ": " << arr[i] << std::endl;
	}
	delete[] arr; // Deallocate the array memory.
	return 0;
}

Explanation of the Code:

  • Single Integer Allocation: The code uses new to allocate memory for a single integer, initializing it with the value 10. The memory is then freed using delete. .
  • Array Allocation : The new operator is used to dynamically allocate an array of integers. After use, the memory is deallocated using delete[] to avoid memory leaks.
  • Pointer Arithmetic with Arrays : The array arr is used with a pointer arrPtr that points to its first element. By incrementing arrPtr , the code accesses subsequent elements of the array without needing explicit indexing.

Tips:

  • Always Pair new with delete : : Ensure that every new operation has a corresponding delete to avoid memory leaks.
  • Use delete[] for Arrays : When freeing dynamically allocated arrays, always use delete[] instead of delete.
  • Nullify Pointers:After deleting a pointer, set it to nullptr to prevent accidentaldereferencing.

Common Mistakes

  • Memory Leaks : Failing to free heap memory results in memory that remains allocated indefinitely, which can degrade performance and crash programs over time.
  • Dangling Pointers : Pointers that reference memory that has already been freed can lead to undefined behavior if accessed again.
  • Stack Overflows : Deep recursion or excessive stack usage can exhaust stack space, leading to a crash.
