Lesson Title: Smart Pointers ( unique_ptr, shared_ptr )

Introduction

Smart pointers in C++ provide automatic and safe management of dynamically allocated memory. Unlike raw pointers, smart pointers ensure proper deallocation, reducing the risk of memory leaks and dangling pointers.

Purpose of the Lesson

The purpose of this lesson is to introduce smart pointers unique_ptr , shared_ptr , and others and explain their advantages over raw pointers. Understanding smart pointers is crucial for writing modern, safe, and efficient C++ code.

Detailed Explanation

  • Smart Pointers Overview : Smart pointers are classes that act like pointers but automatically manage memory.
  • unique_ptr : Manages single ownership of a resource. The resource is automatically deleted when the unique_ptr goes out of scope
  • shared_ptr : Allows multiple pointers to share ownership of a resource. The resource is deleted when the last shared_ptr pointing to it is destroyed

Code Examples


include <iostream>
include <memory> // Include memory header for smart pointers.
int main() {
	// Example of unique_ptr
	std::unique_ptr<int> uniquePtr = std::make_unique<int>(100); // Create a unique_ptr. 
	std::cout << "unique_ptr value: " << uniquePtr << std::endl;

	// unique_ptr cannot be copied, but can be moved
	std::unique_ptr<int> movedPtr = std::move(uniquePtr); // Transfer ownership.

	// Example of shared_ptr
	std::shared_ptr<int> sharedPtr1 = std::make_shared<int>(200); // Create a shared_ptr. 
	std::shared_ptr<int> sharedPtr2 = sharedPtr1; // Copy the shared_ptr (reference count increases).
	std::cout << "shared_ptr value: " << sharedPtr1 << std::endl;
	std::cout << "shared_ptr use count: " << sharedPtr1.use_count() << std::endl; // Shows the 
	number of owners.
	return 0;
}

Explanation of the Code:

  • unique_ptr Example: unique_ptr manages single ownership. It cannot be copied, ensuring exclusive control over the resource. Ownership can be transferred using std::move . .
  • shared_ptr Example : shared_ptr allows multiple pointers to share ownership of the same resource, automatically managing the resource's lifetime through reference counting.

Tips:

  • Prefer Smart Pointers over Raw Pointers : Smart pointers reduce the risk of memory management errors.
  • Use unique_ptr for Exclusive Ownership : Ideal when a resource should have only one owner.
  • Use shared_ptr for Shared Ownership :Suitable when multiple parts of the code need to share a resource

    • Advanced Concepts

    • weak_ptr : A non-owning reference to an object managed by `shared_ptr`, used to prevent circular references
    • Custom Deleters : Allow smart pointers to manage resources other than memory (e.g., file handles).

Common Mistakes

  • Circular References with shared_ptr : Can lead to memory leaks. Use weak_ptr to break cycles
  • Misunderstanding Ownership : Incorrectly managing ownership with smart pointers can cause unexpected behaviors.

    • Real-World Applications

  • Resource Management : Automatically managing the lifetime of dynamically allocated resources
  • Plugin Systems: Managing lifetimes of loaded modules without manual cleanup.
